01

变量与基本类型 Variables & Types

声明变量 Declaring Variables 必须

用 const 声明不会变的值，用 let 声明会变的值。
不要用 var，那是老写法，会带来奇怪的 bug。

// const = 不会再重新赋值的量
const name = "Freya"
const age = 22
const isStudent = true

// let = 之后会改变的量
let score = 0
score = 10  // 可以重新赋值

// 常见类型 / Common types
const text = "字符串 string"
const num = 42           // 数字 number
const flag = false       // 布尔 boolean
const nothing = null    // 空值 null
记住： 默认用 const，只有当你知道这个值之后会变，才换成 let。
数组与对象 Arrays & Objects 必须

数组是有序列表，对象是键值对集合。这两个是 JS 里最常用的数据结构。

// 数组 Array — 有序列表
const fruits = ["苹果", "香蕉", "芒果"]
fruits[0]        // → "苹果"（从0开始）
fruits.length    // → 3

// 对象 Object — 键值对
const user = {
  name: "Freya",
  age: 22,
  isStudent: true
}

user.name      // → "Freya"
user["age"]    // → 22（两种取值方式都可以）
02

函数 Functions

箭头函数 Arrow Functions 必须

现代 JS 最常用的函数写法。你在 React 代码里到处都会看到它。

// 传统写法（了解即可）
function greet(name) {
  return `你好, ${name}!`
}

// 箭头函数写法（常用）
const greet = (name) => {
  return `你好, ${name}!`
}

// 单行可以省略 return 和花括号
const double = (n) => n * 2

greet("Freya")  // → "你好, Freya!"
double(5)       // → 10
模板字符串： 用反引号 ` 包裹，${变量} 直接插入变量，比字符串拼接好用很多。
03

数组操作 Array Methods

三个最常用的数组方法 map / filter / forEach 必须

这三个方法在 React 渲染列表时几乎每天都用。

const nums = [1, 2, 3, 4, 5]

// map — 对每个元素做变换，返回新数组
// "把每个数乘以2"
const doubled = nums.map(n => n * 2)
// → [2, 4, 6, 8, 10]

// filter — 过滤出符合条件的元素
// "只要偶数"
const evens = nums.filter(n => n % 2 === 0)
// → [2, 4]

// forEach — 遍历每个元素，不返回新数组
nums.forEach(n => console.log(n))
// 依次打印 1 2 3 4 5

// 实际场景：React 渲染列表
const users = [{ name: "Freya" }, { name: "Alex" }]
users.map(user => `<li>${user.name}</li>`)
记忆口诀： map 变形，filter 筛选，forEach 遍历。
04

解构赋值 Destructuring

从对象/数组中快速取值 Extract values quickly 必须

现代 JS 和 React 代码里到处都是这个写法，看懂它很重要。

// 对象解构
const user = { name: "Freya", age: 22, city: "Melbourne" }

// 老写法
const name = user.name
const age = user.age

// 解构写法（更简洁）
const { name, age } = user
// name → "Freya", age → 22

// 数组解构
const colors = ["red", "green", "blue"]
const [first, second] = colors
// first → "red", second → "green"

// React useState 就是这个原理！
const [count, setCount] = useState(0)
原来如此： 你在 CampusQuest 里写的 useState 就是数组解构！现在看懂了吧。
05

异步处理 Async / Await

调用 API 必须懂 Essential for API calls 必须

网络请求需要时间，JS 不会傻等——它先去做别的事，等结果回来再处理。async/await 是处理这种"等待"最清晰的写法。

// 调用一个 API 获取数据
const fetchUser = async () => {
  try {
    // await = "等这个结果回来再继续"
    const response = await fetch("https://api.example.com/user")
    const data = await response.json()
    console.log(data)
  } catch (error) {
    // 出错了在这里处理
    console.log("出错了:", error)
  }
}

fetchUser()

// 调用 Claude API 也是同样的写法
const askClaude = async (prompt) => {
  const res = await fetch("/api/claude", {
    method: "POST",
    body: JSON.stringify({ prompt })
  })
  return await res.json()
}
规则： 用了 await，函数前面必须加 async。永远加 try/catch 处理错误。
06

模块系统 Modules

import / export Splitting code into files 必须

代码拆成多个文件，用 export 导出，用 import 引入。React 项目里每个组件都是这样组织的。

// utils.js — 导出函数
export const formatName = (name) => name.toUpperCase()

export default function greet(name) {
  return `Hello, ${name}!`
}

// App.js — 引入使用
import greet, { formatName } from "./utils"

// 引入第三方库（你已经用过）
import { useState } from "react"
import { Html5QrcodeScanner } from "html5-qrcode"
default vs named： export default 导入时可以随便起名；export const 导入时要用花括号 {} 且名字要对应。
